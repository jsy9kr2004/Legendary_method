## 0. 한 줄 요약

분봉 단저단고는 **분류 정확도 (AUC 0.88) 는 좋지만 strict 임계 (top 0.5%) 에서
precision 11.5% 뿐** — 진짜 swing 11.5% / 가짜 swing 88.5% 비율 때문에 평균이
끌어내려진다. 절대 net 양수는 본질 한계.

## 1. 진단 1 — Precision @ top 0.5% = **11.5%** ★ 핵심 약점

```
score top 0.5% 봉수: 556
그 중 sigB_gt=True (진짜 단저): 64 건  (11.5%)
sigB_gt=False (가짜 단저, false positive): 492 건  (88.5%)
```

- AUC 0.88 = "전체 분봉 중 진짜 swing low 봉의 score 가 비-swing 봉보다 높을 확률"
- 그러나 **strict 임계 (top 0.5%) 에서 진짜 swing 비율은 11.5% 뿐**
- AUC 와 precision @ threshold 는 다른 metric — AUC 좋아도 strict 임계에선 가짜가 많음

### 왜 이런 일이 생기나
- 전체 분봉 중 진짜 단저 GT 봉 비율 = 약 0.5%
- score top 0.5% 진입 시 진짜 단저 GT 매칭 = 11.5% (base rate 의 23배 정도, AUC 정보)
- 그러나 절대값으론 여전히 88.5% 가 가짜

## 2. 진단 2 — 진짜 vs 가짜 단저의 성과 차이가 극명

| 케이스 | 건수 | 진입 후 30분 MFE | MAE | 실제 청산 |
|---|---|---|---|---|
| **진짜 단저 (sigB_gt=True)** | 64 | +6.12% | -0.19% | **+3.627%** ✓ |
| **가짜 단저 (sigB_gt=False)** | 492 | +2.32% | -2.19% | **-0.016%** ★ |
| 전체 평균 | 556 | +2.76% | -1.96% | +0.403% |

→ **진짜 단저 64건은 net 매우 좋음** (+3.6%, MAE 거의 0)
→ **가짜 단저 492건은 break-even** (+0%) 인데 **MAE -2.19% 자주 터치**
→ 가짜 단저가 평균을 끌어내림 (64 × 3.6% + 492 × 0% / 556 = +0.40%)

### 약점 정량
- 가짜 단저 492건이 진짜 단저 64건의 좋은 매매 (+3.6%) 를 평균 0.40% 로 희석
- 비용 0.2% 가산 후 net +0.20% → 진입 슬리피지 적용 후 net 음수

## 3. 진단 3 — 봉 close 지연 timing miss

| 케이스 | 다음 봉 close vs sigB 발화 봉 close 슬리피지 |
|---|---|
| 진짜 단저 (sigB_gt=True) | **+0.946%** ★ |
| 가짜 단저 (sigB_gt=False) | +0.054% |

→ **진짜 단저 봉의 다음 봉 가격이 평균 +0.95% 상승** = 봉 close 후 진입 시 진짜
swing low 의 +0.95% 위에서 진입
→ 봉 close 안 기다리고 tick 으로 swing low 회복 시점 진입하면 일부 회복 가능

## 4. 진단 4 — MFE 캐치율 14.6%

```
가능 최대 익절 (평균 MFE) = +2.760%
실제 실현 (평균 realized) = +0.403%
캐치율 = 14.6%
```

→ 가능 swing 의 14.6% 만 실현. 86% 의 swing 놓침.
→ 청산 timing (sigS top 0.5%) 이 진짜 swing high 보다 빠르거나, 진입 후 가격이
swing high 까지 안 가고 빠질 가능성.

## 5. 해결 옵션 비교 + Backtest

### 옵션
| 옵션 | 가설 | 예상 효과 |
|---|---|---|
| A. 임계 더 엄격 (top 0.1%) | 가짜 단저 줄어듦 | 표본 부족 위험 |
| **B. sigB 확실히 발화 봉만** | precision 향상 (가짜 회피) | 가설 |
| **C. 실시간 tick swing 진입** | 봉 close 지연 +0.95% 회복 | 가설 |
| D. sigS 청산 임계 완화 | MFE 캐치율 ↑ | 가설 |

### Backtest (multi=5, floor 2%, in-sample 전체)

| 설정 | n | precision | gross | **net 지정가** | 승률 |
|---|---|---|---|---|---|
| 기준 (top 0.5%, 봉 close) | 556 | 11.5% | +0.478% | +0.278% | 43.7% |
| **B만** | 530 | 12.1% | +0.403% | +0.203% | 42.8% |
| **C만 ★** | 556 | 11.5% | **+0.550%** | **+0.350%** | 45.0% |
| **B+C** | 530 | 12.1% | +0.477% | +0.277% | 44.2% |

### Backtest (train/val 분리, out-of-sample)

| 설정 | n | gross | **net 지정가** | net 유동리더 |
|---|---|---|---|---|
| 기준 | 169 | -0.353% | -0.553% | -0.503% |
| B만 | 168 | -0.343% | -0.543% | -0.493% |
| **C만** | 169 | -0.279% | **-0.479%** | -0.429% |
| **B+C** | 168 | -0.269% | **-0.469%** | -0.419% |

### 발견
1. **B (sigB confirm) 효과 거의 0%p** — score top 0.5% 가 이미 sigB 발화 봉이라
   중복. precision 11.5 → 12.1% 미미 향상.
2. **C (실시간 swing 진입) 효과 +0.07~0.08%p ✓ 일관** — in-sample / out-of-sample
   모두 동일 효과 (overfit 아님).
3. **In-sample net 양수 (+0.28%) vs out-of-sample net 음수 (-0.47%)** — 약 0.75%p
   overfit gap. 표본 부족.
4. **모든 옵션 out-of-sample net 음수** — 단저단고 절대 net 양수 도달은 어려움.

## 6. 본질 한계

```
AUC 0.88 (ranking 좋음)
   ↓
Precision @ top 0.5% = 11.5% (88.5% 가짜)
   ↓
가짜 단저 봉이 평균 끌어내림 (MAE -2.19%)
   ↓
C (실시간 진입) 으로 +0.08%p 회복 가능
   ↓
그러나 out-of-sample 절대 net 음수 (-0.47%) — 본질 한계
```

분봉 단저단고는 분류 정확도 vs 매매 결과의 gap 큼. 핵심 원인:
- 분봉 swing 의 **진짜 vs 가짜 비율 11.5 : 88.5**
- 가짜 단저의 평균 MAE -2.19% (자주 -2% 손절)
- 진입/청산 timing 손실 (MFE 캐치율 14.6%)

## 7. 운영 정책 결정

사용자 결정 (2026-05-29):
- **B + C 둘 다 구현** (코드 변경 작업)
- C 의 실시간 swing 진입을 카드에 표시 — 봉 close 안 기다리고 tick 진입 timing 안내
- B 의 sigB confirm 도 표시 — 사용자 신뢰도 ↑ (효과 0%p 지만 가설 검증 의도)
- 한계 명시 유지 — 사용자 직관 매매 보조 도구, 자동 적용 X

## 8. 향후 개선 가능성

| 영역 | 가능성 |
|---|---|
| 더 큰 표본 (1개월+ 누적) | overfit gap 축소 — true effect 측정 가능 |
| Universe 룰 정정 (회전율 30위 재도입?) | 종목 분포 변경 — 효과 불명 |
| Multi-feature 청산 룰 | MFE 캐치율 14.6 → 30%+ 가능성 |
| 종목별 sigS 임계 튜닝 | 종목별 sweet spot 다를 가능성 |

매매일지 토론 시점 누적 데이터로 재검증 + 한계 재평가.

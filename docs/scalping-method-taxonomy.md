## 1. 매매법 taxonomy

## 2. 매매법별 정량 정의

각 표: ★=통설에 숫자 있음 / △=정성적(우리가 정량화) / ⚠=주의(스윙 수치 등).

### 2.1 돌파매매 (Breakout)

> 상승 추세 종목이 **기간조정 끝**에 **거래량을 동반**해 직전 고가(저항선)를 강하게
> 돌파할 때 따라붙는 기법. (나무위키, brunch @bjbw/9, threads @v_rally55)

| 항목 | 내용 | 등급 |
|---|---|---|
| 진입 대상 | 박스권 상단 / 전고점(직전 고가) 저항선. **박스권 기간이 길수록 상승확률↑** | △ "길수록" 정성 |
| 진입 트리거 | 저항선 돌파 + **평소보다 크게 늘어난 거래량** 동반 | △ "평소보다 크게" — 배수 정량화 필요 |
| 매수 타점 (A형) | 돌파하는 순간 따라붙기 (추격형) | △ |
| 매수 타점 (B형) | 돌파 후 **첫 눌림(되돌림)** 에서 진입 — 통설은 B형을 더 권장 ("돌파 후 바로보다 조정 후 상승이 많다", @bjbw) | △ → 사실상 돌파+눌림 결합 |
| 가짜 돌파 배제 | ① 거래량 부족하면 잠깐 오르고 밀림 → 진입 X. ② **갭상승으로 저항 돌파한 경우 매수 X** (@bjbw) | ★ 갭돌파 배제는 명확 |
| 손절 | **돌파선(저항선) 재이탈** = 돌파 실패 | ★ 기준 명확 (값은 레벨 의존) |
| 익절/목표 | 추세 끝까지 (트레일링). 통설은 명시 수치 빈약 — 사례성 +9% 등만 | △ |
| 보유시간 | 김 (돌파 성공 시 추세 끝까지) | △ |
| 타임프레임 | 일봉 저항 + 분봉(1~5분) 돌파 순간 | △ |
| 시장 조건 | **상승 추세장에서만 성공확률↑.** 지수 하락/악재 시 비중 축소 | ★ regime gate 필수 |

**돌파매매의 핵심 (사용자 함정 직결)**: 진입은 **저항 *레벨* 바로 위**에서 *이르게*.
레벨 한참 위(+5~10%)에서 사면 그건 돌파가 아니라 **추격** — `양봉양봉양봉매수음봉`.
→ 시스템에서 돌파의 "늦음 가드"는 **고점 거리(R14k)가 아니라 돌파 *레벨* 로부터의 거리 +
연장/소진도**로 잡아야 함 (§4.3).

### 2.2 눌림목매매 (Pullback)

> 수급 들어온 종목이 **상승 후 쉬어갈 때(눌릴 때)** 잡는 기법. (한국경제 눌림목 3법칙,
> brunch @hasowon73/57, f-daily, 그래주식)

| 항목 | 내용 | 등급 |
|---|---|---|
| 전제 (1차 상승) | **이전 상승 흐름이 명확** (기준봉 존재) — 그냥 내리는 종목 X | △ "명확" 정성 |
| 지지 확인 | **5일선·10일선 부근 지지** 또는 **과거 매물대**에서 멈춤 | ★ 이평 명시 / △ 매물대 정성 |
| 거래량 패턴 | 상승 시 거래량↑ → **조정 중 거래량 감소**(세력 안 털고 쉬는 신호) → **지지에서 거래량 재유입(반등)** = 진입 | ★ 패턴 명확 / △ 임계 정량화 필요 |
| 되돌림 비율 | 통설 일부 피보나치(38.2/50/61.8%) 언급되나 @hasowon73 은 미명시. 1파 vs 3파 눌림 구분만 | △ 정량화 필요 |
| 매수 타점 | 지지 + 거래량 재유입 확인되는 **반등 초입** | △ |
| 손절 | **5일선 이탈** OR **(기준)양봉 몸통 하단 이탈** | ★ 명확 |
| 익절/목표 | ⚠ 통설: "15% 1차 익절, 30%↑ 분할" — **이건 스윙(수일) 수치.** 오전 단타(#3)는 **직전 고점**을 1차 목표로 스케일 다운해야 (§5 TODO) | ⚠ 스윙 수치 그대로 import 금지 |
| 보유시간 | 짧음(반등~직전 고점). 돌파보다 짧음 | △ |
| 타임프레임 | 일봉 추세 + 분봉 눌림/지지 | △ |

**눌림목 ≠ 떨어지는 칼 받기**: 1차 급등(주도주)한 종목의 *첫 통제된 눌림*이 지지받고
거래량 재유입될 때만. 무지성 하락 매수 = 물타기 (사용자에게 명시).

**눌림 청산의 본질 (사용자 직관 정합)**: 매수세(VP) 한 번 줄어든 것만으론 청산 X — 눌림은
원래 거래량이 줄었다 늘어나는 구조. 청산은 **직전 고점 도달(목표) 분할익절** OR
**지지선(5일선/양봉 하단) 붕괴**. → 현재 시스템의 "단일 청산 시그널 OR" 와 충돌 (§4.2).

### 2.4 VI 매매 (변동성완화장치)

> VI = 순간 변동성 완화 가격안정화 장치. 정적 VI(기준가 대비 ±10% 등)·동적 VI(직전
> 체결 대비). 발동 시 2분 단일가. (KB think, KRX)

- 단타에서 VI 발동은 **독립 매매법이라기보다 변곡 트리거**. 상승 VI 발동 = 강한 모멘텀
  신호(돌파의 강화) / 발동 해제 직후 흐름이 갈림.
- → **돌파매매의 보조 시그널**로 흡수 (VI 발동 = 돌파 확정 가산 후보). 별도 매매법으로
  분리 X (v0).
- 우리 시스템엔 이미 Exit.Triggers E5(VI) 존재 → 청산 쪽엔 반영돼 있음. 진입 가산은 △ TODO.

## 4. 시스템 매핑 (현 Buy.Score / Exit.Triggers 대비)

### 4.1 매매법별 진입 시그널 — 있는 것 / 없는 것

| 매매법 | 현 Buy.Score 로 커버 | 추가 필요 |
|---|---|---|
| 돌파 | 회전율/거래대금, VP, 거래량비율, 가속 | **저항선(전고점/박스상단) 돌파 감지**, **돌파 레벨 거리**, 박스 기간, VI 발동, 갭돌파 배제 |
| 눌림 | VP, 5MA/20MA 위치(`price_vs_ma5`), VWAP | **1차 급등(기준봉) 식별**, **조정 중 거래량 감소→재유입 패턴**, 5일선 지지 판정, 직전 고점(목표) |

→ 현 Buy.Score 는 모멘텀/위치 짬뽕이라 **두 매매법 어느 쪽의 *셋업 구조*(저항돌파 / 눌림지지)도
명시적으로 못 잡음.** 셋업 감지 로직이 신규 필요.

### 4.2 매매법별 청산 세트 lock (§8.2 #1 구현)

| 매매법 | 청산 트리거 (이 매매법 것만 표시) |
|---|---|
| **돌파** | **빠르게** — 돌파 레벨 재이탈 / 모멘텀 사망(VP 급감, 현 E1/E3) / 윗꼬리 음봉(E4) / VI(E5). *매수세 감소가 1순위 탈출* |
| **눌림** | **느리게** — 직전 고점(목표) 도달 분할익절 / 5일선·양봉하단(지지) 붕괴. *VP 일시 감소만으론 청산 X* |

→ `/buy CODE PRICE` 에 **진입 매매법 인자** 추가(`/buy CODE PRICE breakout|pullback`),
tick_log 에 박고, 카드는 해당 매매법 청산 세트만 surface. (현재 "단일 시그널 OR 즉시 청산"
= 눌림 진입에 돌파 청산 적용하는 불일치 → 10초 보유의 원인.)

### 4.3 R14k(고점 회피) 매매법별 처리 (§8.2 #2 구현)

| 매매법 | R14k 처리 |
|---|---|
| **눌림** | **유지.** 올바른 눌림 진입은 고점 아닌 눌린 자리 → 안 걸림. 반등 고점 추격 방지 효과. |
| **돌파** | **교체.** 고점 거리 대신 → ① 돌파 *레벨* 로부터 거리(+0~1% 좋음, +5~10% 추격) ② 박스 다지기 유무 ③ 연장/소진도(연속 양봉·최근 5분 폭등률 = R14e/R14g). |

### 4.4 셋업 분류기 초안 (보수적 — §8.2 #5)

매 tick 라벨 `{breakout, pullback, none}` + confidence. **자신 없으면 `none` 후하게.**

- `breakout` 후보: 직전 N분 박스/전고점 존재 + 현재가 그 레벨 돌파 + 거래량 동반 +
  레벨 거리 작음.
- `pullback` 후보: 직전 기준봉(20억+) 급등 존재 + 이후 조정(거래량 감소) + 5일선/직전봉
  하단 부근 지지 + 거래량 재유입 시작.
- 둘 다 약하거나 애매 → `none`. (로깅은 항상, surface 는 confident 일 때만.)

**임계 숫자는 본 문서에 박지 않음** — §5 데이터 검증으로 결정.

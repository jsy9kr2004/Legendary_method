## 진단

기존 backtest (v1) 의 cutoff 변경 (5→6 또는 5→4) 은 사용자 의도 (정점 회피) 와 무관 — 본질은 점수 임계가 아니라 **lagging indicator 동시 발화 = 정점 신호** 구조적 함정 (`docs/buy-score-revision-proposal.md` §3.1~3.2). 진짜 해결 = 정점 회피 페널티 시그널 신설.

## 신규 시그널 (proposal P1 + 사용자 의도 정합 추가)

### proposal P1 (lagging 보완)

- **R14e**: 최근 5분 +10% → -2, +5% → -1 (폭등 막바지)
- **R14f**: BB 상한 도달 (≥0) → -1, 돌파 (≥+1%) → -2 (Bollinger 평균회귀)
- **R14g**: 직전 5봉 중 양봉 4+ → -1 (연속 양봉 추격)
- **R14h**: 거래량 정점 + 가격 정체 → -1.5 (매도 출회)

### ★ 사용자 의도 정합 추가 (5/20 매매 분석 후)

- **R14k**: 일중 최고점 거리 — `dist >= -2%` → -2, `dist >= -5%` → -1
- **R14l**: 횡보 정점 — `daily_return >= 15% AND dist >= -5%` → -1.5 (수젠텍 케이스)

R14k/R14l 은 사용자가 "고점" 으로 인지한 케이스 — 일중 상승률 + 정점 절대 위치 — 를 직접 잡음. R14e/f/g/h 는 폭등 속도/막바지 잡지만 횡보 구간 정점 진입은 못 잡음.

| 주성엔 | 09:01:02 | 09:01:04 (2초 차) | 시초 → 첫 STRONG 그대로 | +1.10% → +0.11% (청산 차이) |
| 현대모비스 | 10:09:00 | 10:09:00 | 페널티 발화 X (1분봉 부족) | 0% → 0% |

**본 시뮬의 한계 — 사용자 의도와의 gap**:
- 종목당 첫 STRONG = 시초 1~2분 → 정점 페널티 시점에 1분봉 부족 → 페널티 효과 X
- 사용자가 "고점 매수" 라고 인지한 케이스는 **첫 진입이 아니라 N차 재진입**
- 본 backtest 는 "감시 모드 simulation" — 사용자 운영 시 같은 종목 재매수 패턴 측정 불가

→ **시뮬 1 (사용자 매매 차단)** 이 진짜 측정. p1_plus_kl 7건 차단 = 사용자가 "고점" 매수한 케이스 정확 식별.

## 결론

### 진짜 효과

**p1_plus_kl (R14e + R14f + R14g + R14h + R14k + R14l)** 가 사용자 의도 정합:

1. **시뮬 1 (사용자 매매 차단)**: 7/15 차단, 누적 +5.24% → **+12.21%**
2. **시뮬 2 (전체 backtest)**: avg_entry_5m **+1.48%** (가장 낮음 — 폭등 초기 진입), 평균 +0.28% (current 대비 +56%), 승률 +5.1%p
3. **시뮬 3 (image 차트)**: 수젠텍 1건만 시점 이동 (09:16→09:46, +30분). 다른 3건은 본 시뮬 한계로 측정 불가.

### 거래비용 차감 후 (정직)

- 평균 +0.28% < 왕복 거래비용 0.4~0.5% — **차감 후 break-even 또는 -0.1% 적자**
- 5/18, 5/19 universe 편향 + N=51 표본 작음
- **운영 적용 정당화 불충분**

### 그러나 사용자 매매 패턴 적용 시:

5/20 사용자 실제 매매 (재진입 포함) 에 페널티 적용 시:
- 7건 차단 (5건이 손실 매매, 2건이 미세익)
- 차단 후 누적 +5.24% → +12.21% (수익률 133% 증가)
- 거래비용 차감 — 15건 → 8건 (왕복 비용 7건 × 0.4% = 2.8% 절약)
- 차감 반영 시 실제 효과: +12.21% - 0건 × 0.4% (이미 안한 매매라 비용 X) = **+12.21% 유지**

→ **본 backtest 의 한계를 보완한 시뮬 1 결과가 진짜 가치**. 사용자가 같은 종목 N차 재진입 패턴이라면 R14k/R14l 페널티 큰 효과.
